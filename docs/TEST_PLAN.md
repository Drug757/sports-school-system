# План тестирования Sports School

## Цель тестирования

Проверка корректности работы веб-приложения Sports School после развертывания на платформе PythonAnywhere.

## Область тестирования

Проверяются следующие компоненты системы:

* Главная страница
* Управление группами
* Управление пользователями
* Управление занятиями
* Учёт посещаемости
* Навигация между разделами

## Тестовые сценарии

| ID    | Сценарий                            | Ожидаемый результат              |
| ----- | ----------------------------------- | -------------------------------- |
| TS-01 | Открытие главной страницы           | Страница успешно загружается     |
| TS-02 | Переход в раздел «Группы»           | Отображается список групп        |
| TS-03 | Создание новой группы               | Группа сохраняется в БД          |
| TS-04 | Создание пользователя               | Пользователь появляется в списке |
| TS-05 | Создание занятия                    | Занятие успешно создаётся        |
| TS-06 | Просмотр посещаемости               | Данные отображаются корректно    |
| TS-07 | Переход по навигационному меню      | Все ссылки работают              |
| TS-08 | Обращение к несуществующей странице | Возвращается ошибка 404          |

## Критерии успешного прохождения

Тестирование считается успешным, если все основные сценарии выполняются без критических ошибок.
