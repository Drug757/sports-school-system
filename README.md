# Sports School System

Веб-приложение для управления спортивной школой, разработанное на Django.

## Описание проекта

Система предназначена для автоматизации базовых процессов спортивной школы:

* управление пользователями;
* управление спортивными группами;
* планирование занятий;
* учёт посещаемости;
* просмотр данных через веб-интерфейс.

Проект разработан в рамках учебной практики по дисциплине разработки программного обеспечения.

---

## Основные возможности

### Пользователи

Система поддерживает несколько ролей:

* Администратор
* Тренер
* Участник

Для каждого пользователя сохраняются:

* имя;
* роль.

### Группы

Поддерживаются:

* создание групп;
* назначение тренера;
* добавление участников;
* просмотр списка групп.

### Расписание

Поддерживаются:

* создание занятий;
* привязка занятия к группе;
* указание даты и времени;
* просмотр расписания.

### Посещаемость

Поддерживаются:

* отметка присутствия участников;
* просмотр журнала посещаемости.

## Установка и запуск

### Клонирование репозитория

```
git clone https://github.com/Drug757/sports-school-system.git
cd sports-school-system
```

### Создание виртуального окружения

```
python -m venv .venv
```

Windows:

```
.venv\Scripts\activate
```

Linux:

```
source .venv/bin/activate
```

### Установка зависимостей

```
pip install -r requirements.txt
```

### Применение миграций

```
python manage.py migrate
```

### Запуск проекта

```
python manage.py runserver
```

После запуска приложение будет доступно по адресу:

```
http://127.0.0.1:8000
```

## Автоматизация

Для упрощения работы используются bat-скрипты:

### Основные

* setup.bat — подготовка рабочего окружения проекта
* run.bat — запуск приложения
* check.bat — проверка конфигурации Django
* test.bat — запуск базовой проверки работоспособности проекта
* format.bat — проверка и форматирование исходного кода

### Тестирование и качество

* api-test.bat — выполнение проверки API приложения
* quality-check.bat — комплексная проверка качества проекта
* performance.bat — анализ производительности приложения

### Безопасность

* security-check.bat — поиск потенциально опасных данных и проверка безопасности проекта
* deps-check.bat — проверка зависимостей на наличие известных уязвимостей
* ports-check.bat — просмотр активных сетевых подключений и открытых портов
* logs-security.bat — анализ журналов приложения на наличие ошибок и подозрительных событий

### Резервное копирование

* backup.bat — создание резервной копии базы данных
* restore.bat — восстановление данных из резервной копии

### Релизы

* build.bat — подготовка проекта к выпуску
* release-check.bat — проверка готовности проекта к релизу
* create-release.bat — создание релизной версии проекта

### Дополнительные

* logs.bat — просмотр журналов приложения
* docker-up.bat — запуск проекта в Docker
* docker-down.bat — остановка Docker-контейнеров
